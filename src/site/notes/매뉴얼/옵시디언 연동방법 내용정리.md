# 옵시디언 연동방법 내용정리

> [[매뉴얼/매뉴얼 인덱스\|← 매뉴얼 인덱스로]]

> 이 문서는 Obsidian 볼트를 GitHub → Digital Garden → Vercel로 배포하는 자동화 파이프라인 구성 방법과,  
> 팀원들이 공동으로 참여하는 협업 연동 방법을 정리한 가이드입니다.

---

## 1. 현재 구조

```
도연님 Obsidian 볼트
  └─ Digital Garden 플러그인
        └─ GitHub repo (ehdusdk/my-digital-garden)
              └─ Vercel 자동 배포
                    └─ https://my-digital-garden-seven-umber.vercel.app
```

- **Obsidian**: 로컬 마크다운 편집 환경 (노트북)
- **Digital Garden 플러그인** (v2.78.0, Ole Eskild Steensen): `dg-publish: true` 설정된 노트를 GitHub으로 푸시
- **GitHub**: `ehdusdk/my-digital-garden` 저장소 — 노트 소스 관리
- **Vercel**: GitHub 변경 감지 시 자동 빌드 및 배포

---

## 2. 노트 발행 기본 조건

노트가 사이트에 게시되려면 frontmatter에 반드시 아래 항목 포함:

```yaml
---
dg-publish: true
---
```

발행 후 Obsidian 리본(사이드바)의 🌿 Digital Garden 아이콘 클릭 → **Publish Multiple Notes** 실행

---

## 3. Digital Garden 플러그인 핵심 설정

| 설정 항목 | 값 | 비고 |
|-----------|-----|------|
| `GitHub repo name` | `my-digital-garden` | 저장소 이름 |
| `GitHub username` | `ehdusdk` | 저장소 소유자 |
| `GitHub token` | `github_pat_...` | Fine-grained PAT (만료 주의) |
| `Base URL` | `https://my-digital-garden-seven-umber.vercel.app` | **반드시 입력** — 비어있으면 내부 링크 깨짐 |

> ⚠️ **`Base URL` 미설정 시 발생 증상:**  
> 내부 링크 URL이 `https://노트제목/` 처럼 도메인 없이 생성되어 링크 클릭 시 404 오류 발생

---

## 4. GitHub PAT 관리

### PAT 발급 방법

1. [https://github.com/settings/tokens](https://github.com/settings/tokens) 접속
2. **Fine-grained personal access tokens** → **Generate new token**
3. 설정:

| 항목 | 설정값 |
|------|--------|
| Token name | `obsidian-digital-garden` |
| Expiration | `90일` 또는 `1년` |
| Repository access | `Only select repositories` → `ehdusdk/my-digital-garden` |
| Permissions - Contents | `Read and write` |
| Permissions - Pull requests | `Read and write` |

4. 발급된 `github_pat_...` 값을 Obsidian Digital Garden 설정에 입력

> ⚠️ **보안 주의사항**: PAT는 플러그인 설정 파일(`data.json`)에 평문 저장됨  
> 만료 시 또는 노출 의심 시 즉시 삭제 후 재발급 필요

---

## 5. 팀원 협업 연동 방법

팀원들도 같은 사이트에 노트를 발행하려면 아래 3가지 방식 중 선택.

---

### 방식 1 — 팀원 각자 Obsidian + Digital Garden 설정 (권장)

```
김도연 Obsidian ──┐
팀원A Obsidian ───┼──→ ehdusdk/my-digital-garden (GitHub) ──→ Vercel
팀원B Obsidian ──┘
```

**도연님이 할 일 (1회):**

1. GitHub repo → **Settings → Collaborators → Add people**  
   → 팀원 GitHub 계정을 `Write` 권한으로 초대
2. 노트 폴더 구조 사전 약속 (파일 충돌 방지):

```
src/site/notes/
├── 김도연/       ← 도연님 노트
├── 홍길동/       ← 팀원A 노트
├── 이영희/       ← 팀원B 노트
└── 공통/         ← 공동 회의록, 공지
```

**각 팀원이 할 일:**

1. Obsidian 설치
2. Community Plugins → **Digital Garden** 설치 및 활성화
3. 플러그인 설정:

| 항목 | 설정값 |
|------|--------|
| GitHub repo name | `my-digital-garden` |
| GitHub username | `ehdusdk` (저장소 소유자 ID) |
| GitHub token | 본인 계정으로 발급한 PAT |
| Base URL | `https://my-digital-garden-seven-umber.vercel.app` |

4. 노트에 `dg-publish: true` 추가 후 **Publish Multiple Notes** 실행

---

### 방식 2 — GitHub 웹에서 직접 MD 파일 업로드 (가장 간단)

Obsidian 없이 GitHub 웹 UI에서 직접 `.md` 파일 업로드.

**절차:**

1. 팀원을 repo Collaborator로 초대 (방식 1과 동일)
2. GitHub 웹 → `src/site/notes/팀원이름/` 폴더에 파일 업로드
3. 파일 상단에 반드시 포함:

```yaml
---
dg-publish: true
---
# 제목
내용...
```

4. Commit → Vercel 자동 배포 (약 1~2분 소요)

---

### 방식 3 — Obsidian 볼트 공유 + 클라우드 동기화

OneDrive/SharePoint로 볼트 폴더를 팀 전체 공유.

```
OneDrive 공유 폴더 (볼트)
  ↓ 각자 PC에서 자동 동기화
팀원 각자 Obsidian (동일한 볼트)
  ↓ 발행 담당자 1명이 Digital Garden으로 발행
GitHub → Vercel
```

> ⚠️ 동시 편집 시 파일 충돌 가능성 있음. 발행은 1명이 담당하는 구조.

---

## 6. 방식 비교

| 항목 | 방식 1 (각자 Obsidian) | 방식 2 (GitHub 직접) | 방식 3 (볼트 공유) |
|------|----------------------|--------------------|--------------------|
| 설정 난이도 | 중간 | 쉬움 | 쉬움 |
| Obsidian 필요 여부 | 각자 설치 | 불필요 | 각자 설치 |
| GitHub 계정 필요 | 각자 필요 | 각자 필요 | 1명만 필요 |
| 자율 발행 | 각자 가능 | 각자 가능 | 담당자 1명 |
| 노트 간 wikilink 연결 | 수동 | 수동 | 자동 연결 |
| 충돌 위험 | 낮음 | 낮음 | 중간 |
| **추천 상황** | IT팀 내부 협업 | 빠른 시작 | 긴밀한 팀 협업 |

---

## 7. 트러블슈팅 — 업로드가 안 될 때 체크리스트

```
□ 1. GitHub PAT 만료 여부 확인 → 재발급 후 플러그인 재설정
□ 2. Base URL 입력 여부 확인 (비어있으면 내부 링크 깨짐)
□ 3. publishPlatform 설정 확인 (SelfHosted → 기본값으로 변경)
□ 4. 노트 frontmatter에 dg-publish: true 있는지 확인
□ 5. GitHub repo에 Collaborator 권한 있는지 확인
□ 6. Vercel 배포 상태 확인: https://github.com/ehdusdk/my-digital-garden/actions
```
